STOCKALERT — V8.0.01

Objectif du sprint
- Démarrer le chantier KPI Backtest + Games de façon SAFE.
- Introduire un socle commun de calcul sans grosse refacto.
- Réduire la duplication entre Backtest et Games sur les agrégats partagés.

Revue de code préalable
- Les KPI Backtest portefeuille étaient calculés inline dans engine.py.
- Les Games utilisaient run_backtest_kpi_only() mais sans agrégats globaux partagés.
- Les concepts "ticker joué", "RATIO_IN_POSITION", "BMD", gains/pertes et win-rate n'avaient pas encore de module commun.
- Risque principal : divergence progressive entre Backtest et Games.

Corrections livrées
1) Nouveau module commun KPI
- Ajout de core/kpi/common.py
- Helpers purs, déterministes et réutilisables :
  - to_decimal
  - compute_total_return
  - compute_daily_average
  - compute_presence_ratio
  - aggregate_played_ticker_stats
  - aggregate_amount_stats_from_ticker_entries

2) Backtest
- engine.py utilise désormais les helpers communs pour :
  - BT / BMJ portefeuille
  - NB_PLAYED_TICKERS
  - agrégats played / positive BMD / non-positive BMD
  - agrégats montants (gains, pertes, win/loss trades, profit factor, max gain/loss)
- Aucun changement de structure de sortie JSON.
- Aucun changement de modèle.

3) Games
- runner.py utilise maintenant compute_presence_ratio pour RATIO_IN_POSITION.
- today_results embarque désormais un bloc :
  - kpi_common
- Ce bloc vient du même helper aggregate_played_ticker_stats que Backtest.
- Cela prépare l'harmonisation progressive des KPI globaux entre Backtest et Games.

Tests ajoutés/renforcés
- test_aggregate_played_ticker_stats_counts_only_played_tickers
- test_aggregate_amount_stats_from_ticker_entries_sums_amount_fields
- test_game_runner_updates_snapshot_and_interprets_thresholds_in_percent enrichi pour vérifier today_results.kpi_common.NB_PLAYED_TICKERS

Impact / compatibilité
- SAFE, additif, sans migration.
- Pas de refacto brutale du moteur.
- Pas de changement du contrat principal des résultats Backtest.
- Games obtient seulement un enrichissement JSON supplémentaire.
