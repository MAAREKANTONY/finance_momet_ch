Aide exports

Cette page documente les exports disponibles, leur contenu, leur format et les causes de plantage les plus probables.

Règle pratique : privilégier les exports async visibles dans Traitements. Les exports Excel sont plus lourds que les CSV/Parquet et peuvent être tronqués volontairement sur les très gros backtests.

1. Page Alertes

ExportFormatContenuNotes
Export Alerts CSV CSV async Date, scénario, ticker, exchange, codes d'alerte. Le plus léger. Adapté aux filtres par date, scénario, ticker et codes.

2. Page Alertes / Data exports

ExportFormatContenuNotes
Excel: scénario XLSX async Un onglet par ticker : barres journalières, métriques du scénario, alertes. Peut devenir lourd si beaucoup de tickers et beaucoup d'historique.
ZIP (tous scénarios) ZIP async contenant plusieurs XLSX Un fichier Excel par scénario. Le plus coûteux sur cette page : multiplication par le nombre de scénarios.
Export data XLSX XLSX async 3 onglets : Bars, Metrics, Alerts. Export transversal legacy. Peut être volumineux si le filtre est trop large.

3. Page Backtest

ExportFormatContenuNotes
Exporter Excel XLSX async Settings, Universe, Summary, Portfolio, Portfolio_Daily, un onglet détaillé par ticker/ligne sélectionné. Sur gros univers, les détails peuvent être limités au Top N pour éviter les fichiers trop lourds.
Exporter Excel (compact Sheets) XLSX async Version plus compacte avec feuille Daily consolidée et éventuellement Charts. Souvent préférable pour les gros backtests.
Exporter Debug Excel XLSX async Debug d'un couple ticker / ligne : paramètres, signaux, daily détaillé. Ciblé diagnostic. À utiliser sur un ticker précis.
Exporter Debug CSV CSV async Le daily brut d'un couple ticker / ligne. Le plus simple pour analyse externe.
Export détails ZIP de Parquet ou ZIP de CSV Un fichier par ticker/ligne stocké en Parquet lors du backtest. Option CSV disponible à partir de ces Parquet. Nécessite ENABLE_PARQUET_STORAGE=1 au moment du backtest.

4. Page Jeu

ExportFormatContenuNotes
Export XLS XLSX async Onglet Game avec les paramètres, onglet TodayResults avec les résultats du dernier run. Ne contient pas tout l'historique du game, surtout le snapshot courant.

Causes probables de plantage

  • Volume trop important : trop de tickers, trop d'historique, trop de scénarios, ou charts générés sur un gros backtest.
  • Timeout / mémoire : un export sync ou trop lourd peut épuiser la RAM ou dépasser le temps disponible.
  • Données imbriquées : certains champs issus des résultats JSON / Parquet peuvent contenir des listes ou structures non directement exportables en CSV/Excel.
  • Parquet absent : l'export détails dépend des fichiers écrits au moment du backtest.
  • Filtres trop larges : export sans ticker, sans exchange, sans période.

Recommandations

  • Commencer par un export ciblé sur un ticker ou une période courte.
  • Pour les gros backtests, préférer Excel compact ou Parquet.
  • Réserver les exports debug à l'analyse d'un seul ticker / ligne.
  • En cas d'échec, vérifier le job dans la page Traitements.
