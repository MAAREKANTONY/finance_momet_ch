STOCKALERT — V7.0.82

Iteration 3 — audit_jobs

What was added
- New management command: python manage.py audit_jobs
- Read-only job audit helpers in core/job_recovery.py
- Console summary with healthy / suspect / critical counts
- JSON output support (--json)
- Filtering by ids (--ids)
- Optional shortcut threshold (--stale-minutes)
- Optional exclusion of pending jobs (--no-pending)
- Documentation update in docs/JOBS_ROBUSTNESS.md
- Targeted tests added in core/tests/test_jobs_robustness.py

What audit_jobs detects
- RUNNING jobs with stale heartbeat
- RUNNING jobs with no heartbeat for too long
- RUNNING jobs still active after cancel / kill request
- PENDING jobs that are too old
- PENDING jobs still queued after cancel / kill request

Operational usage
- Diagnose first:
  docker compose exec web python manage.py audit_jobs
- JSON export:
  docker compose exec web python manage.py audit_jobs --json
- Then recover if needed:
  docker compose exec web python manage.py recover_jobs

Notes
- No migration in this iteration
- Read-only command: audit_jobs does not change DB state
- Python compilation of changed files passes in the container
