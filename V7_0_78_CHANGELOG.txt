V7.0.78 — Reset signal memory after executed trades

Scope
- Fix stale signal memory causing immediate re-buy / re-sell after an executed opposite trade.
- Applies to backtests and games because both use core/services/backtesting/engine.py.

Changes
- Added helper _reset_trade_signal_memory(state_row) in core/services/backtesting/engine.py.
- After every executed SELL:
  - clear active_signal_states
  - clear and_latched_states
- After every executed BUY:
  - clear active_signal_states
  - clear and_latched_states
- Added regression test ensuring a SELL does not trigger an immediate same-day re-BUY from stale AND memory.

Notes
- This is intentionally simple and SAFE: full reset of persisted signal memory on executed trade.
- No change to OR/AND semantics themselves.
- No change to GM filter semantics.
