V8.0.08d HOTFIX

Scope
- Minimal targeted fix for launch guard stale recovery.
- No Celery/Redis/UI/KPI changes.
- No recovery-threshold changes.

Root cause
- The previous launch path tried to run a broader stale recovery before launch.
- That was larger than needed for the failing test and could interfere with other job-recovery expectations.

Fix applied
- launch_processing_job() now:
  1. queries the current active job,
  2. if that exact active job is obviously stale, applies inline recovery only to that job,
  3. re-queries the DB,
  4. raises conflict only if an active job still exists.
