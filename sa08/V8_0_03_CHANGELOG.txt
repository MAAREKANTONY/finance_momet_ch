V8.0.03 — Single queue hardening

Goal
- Enforce the simplest possible execution model: only one tracked job active at a time.

Changes
- global launch guard in core.job_launch: refuse every new tracked launch while any PENDING/RUNNING ProcessingJob exists
- best-effort inline recovery before rejecting a launch, to unblock obvious zombies immediately
- conflict launches now create a FAILED ProcessingJob row with an explicit error instead of silently stacking jobs
- default stale recovery thresholds shortened for the single-queue operating mode:
  - heartbeat: 2 min
  - started/no-heartbeat: 3 min
  - pending: 10 min
  - requested stop: 1 min
- docs/JOBS_ROBUSTNESS.md updated

Tests added
- block launch when any active job already exists
- allow launch again after inline recovery of a stale active job
- recover stale RUNNING jobs quickly
- recover stale PENDING jobs quickly

No migration.
