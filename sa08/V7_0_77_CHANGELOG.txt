V7.0.77 — Fix legacy CSV normalization for backtest/game signal codes

Root cause
- The backtest/game engine normalized string signal codes as a single token.
- Legacy stored data could contain CSV strings like "SPV,SPVV,SPV_BASSE,SPVV_BASSE".
- In OR logic, this was interpreted as one literal code instead of four codes.
- Result: sell conditions could fail to trigger even when alerts were present.

Changes
- Updated core/services/backtesting/engine.py::_normalize_codes
  - splits comma-separated strings into distinct codes
  - supports mixed list payloads that may themselves contain CSV items
  - trims whitespace
  - uppercases values
  - deduplicates while preserving order
- Added regression tests in core/tests/test_engine_and_metrics.py for:
  - legacy CSV sell OR logic
  - mixed list + CSV AND logic with latched memory

Scope
- Backtests
- Games (same engine logic)

Risk
- Low and targeted.
- No change to the OR/AND operator semantics themselves.
- Only input normalization was broadened to accept legacy storage formats.
