V7.0.81 — Jobs hardening iteration 4 (recover_jobs)

Main goal
- Add a real operational recovery command for stale/zombie async jobs.

Delivered
- New helper: core/job_recovery.py
- New command: python manage.py recover_jobs
- Recovery rules based on stale RUNNING/PENDING jobs
- Automatic terminal status selection:
  - KILLED if kill_requested=True
  - CANCELLED if cancel_requested=True
  - FAILED otherwise
- Related object sync preserved via sync_related_state_for_terminal_job()
- Documentation updated in docs/JOBS_ROBUSTNESS.md
- Targeted tests added for recover_jobs

Operational usage
- Standard recovery:
  docker compose exec web python manage.py recover_jobs

- Dry-run:
  docker compose exec web python manage.py recover_jobs --dry-run

- Explicit ids:
  docker compose exec web python manage.py recover_jobs --ids 12,15

Notes
- No database migration in this iteration.
- This iteration focuses on DB/business recovery, not on worker process inspection.
