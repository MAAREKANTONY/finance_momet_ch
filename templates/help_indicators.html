Aide — indicateurs, signaux, variables et Games

Cette page est la référence fonctionnelle du moteur. Elle regroupe les variables de scénario, les indicateurs calculés, les signaux d'achat / vente, ainsi que les métriques spécifiques aux Games. Sauf mention contraire, le prix utilisé est le prix composite P(t).

1) Prix composite P(t)

Inputs (DailyBar) : F = Close, H = High, L = Low, O = Open

Paramètres scénario : a, b, c, d

P(t) = (a·F(t) + b·H(t) + c·L(t) + d·O(t)) / (a+b+c+d)

C'est ce prix P(t) qui est utilisé dans les calculs des indicateurs legacy, des lignes flottantes et des signaux de pente.

2) Rendement journalier 
δj(t) = (P(t) - P(t-1)) / P(t-1)

Utilisé par :

  • K2f
  • Kf2bis
  • Kf3
  • SUM_SLOPE / SPa / SPv

4) Famille legacy — M, X, M1, X1, T, Q, S

Fenêtres : N1 et N2

M(t)  = max(P(t-1), ..., P(t-N1))
X(t)  = min(P(t-1), ..., P(t-N1))
M1(t) = moyenne sur N2 jours de M(t)
X1(t) = moyenne sur N2 jours de X(t)
T(t)  = (M1(t) - X1(t)) / e
Q(t)  = M1(t) - T(t)
S(t)  = M1(t) + T(t)

Cette famille sert aux indicateurs K1, K2, K3, K4 et K1f. Elle est distincte des grandeurs flottantes Mf1 / Xf1 utilisées dans K2f, Kf2bis et Kf3.

5) Indicateurs legacy — K1, K2, K3, K4 
K1(t) = P(t) - M1(t)
K2(t) = P(t) - X1(t)
K3(t) = P(t) - Q(t)
K4(t) = P(t) - S(t)
6) K1f 
E(t)       = M1(t) - X1(t)
ratio_p(t) = RATIO_P(t) / 100
C(t)       = (VC - ratio_p(t)) · FL · E(t)
K1f(t)     = K1(t) + C(t)

K1f ajoute une correction dépendante de RATIO_P, VC et FL à la ligne K1.

7) Indicateurs de tendance — V, slope_P, nb_pos_P, sum_pos_P, RATIO_P, AMP_H
  • V : variation journalière du close vs close veille, exprimée en %.
  • slope_P : moyenne des variations journalières sur N3 jours.
  • nb_pos_P : nombre de slope_P positifs sur N4 jours.
  • sum_pos_P : somme des slope_P positifs sur N4 jours.
  • RATIO_P : nb_pos_P / N4 × 100.
  • AMP_H : intensité moyenne des hausses dérivée de sum_pos_P, nb_pos_P et N3.
8) K2f — ligne flottante 2

Fenêtres / paramètres : N1, N2, N5, K2J, cr, e

slope1(t)     = moyenne sur N1 jours de δj(t)
slope2(t)     = moyenne sur N2 jours de δj(t)
slope_deg(t)  = slope1(t) - slope2(t)
h             = max(1, floor(N5/2))
Mf1(t)        = moyenne sur h jours des max de P sur des fenêtres glissantes de taille N5
Xf1(t)        = moyenne sur h jours des min de P sur des fenêtres glissantes de taille N5
Ef(t)         = Mf1(t) - Xf1(t)
K2f(t)        = Mf1(t) - slope_deg(t) · cr · Ef(t) / e

K2f est la ligne flottante 2 actuellement utilisée pour les signaux AF / BF.

9) Kf2bis — ligne flottante 2 bis

Fenêtre / paramètres : N2, N5, e

p(t)      = somme sur N2 jours de δj(t)
Ef(t)     = Mf1(t) - Xf1(t)
Kf2bis(t) = Mf1(t) - Ef(t) · p(t)

Mf1, Xf1 et Ef sont les mêmes grandeurs locales que pour K2f.

10) Kf3 — ligne flottante 3

Paramètres : N5f3, crf3, NampL3, baseL3, periodeL3, e

r(t)            = (P(t) - P(t-1)) / P(t-1)
amp(t)          = moyenne sur NampL3 jours de |r(t)|
k(t)            = amp(t) / baseL3
periode(t)      = periodeL3 · baseL3 / amp(t)
slope_degL3(t)  = moyenne de r(t) sur periode(t)
Mf1L3(t)        = moyenne locale des max de P sur N5f3
Xf1L3(t)        = moyenne locale des min de P sur N5f3
EfL3(t)         = Mf1L3(t) - Xf1L3(t)
Kf3(t)          = Mf1L3(t) - slope_degL3(t) · crf3 · EfL3(t) / e
11) Ligne V basée sur les plus hauts 
M              = m_v
M1             = max(1, floor(M/2))
V_pre(t)       = max(High) sur M jours
V_line(t)      = moyenne mobile sur M1 jours de V_pre

La ligne V compare les High du marché à une ligne de résistance lissée.

12) SUM_SLOPE et signaux SPa / SPv

Paramètres : Npente, Seuil de pente

SUM_SLOPE(t) = somme sur Npente jours de δj(t)

Déclencheur achat SPa :

SUM_SLOPE(t-1) < Seuil_de_pente
et
SUM_SLOPE(t)   > Seuil_de_pente

Déclencheur vente SPv :

SUM_SLOPE(t-1) > Seuil_de_pente
et
SUM_SLOPE(t)   < Seuil_de_pente

Ces signaux sont disponibles dans les backtests, alertes et Games.

14) Où les signaux sont utilisés
  • Alertes : filtrage et envoi des alertes quotidiennes.
  • Backtests : BUY / SELL sélectionnent un code de signal.
  • Games : les calculs journaliers utilisent les mêmes métriques et signaux de base.
15) Variables spécifiques aux Games
Variable Définition
BMDPerformance issue du backtest / score de tradabilité.
TRADABLE_DAYSNombre de jours où le ticker est considéré tradable.
TRADABLE_DAYS_IN_POSITION_CLOSEDNombre de jours tradables terminés en position fermée.
RATIO_IN_POSITIONPourcentage du temps où le ticker est en position.
SUM_SLOPEMomentum cumulé sur Npente jours, utilisé aussi pour SPa / SPv.
tradability_thresholdSeuil minimal de BMD pour considérer le ticker tradable.
slope_thresholdSeuil minimal de SUM_SLOPE pour la tradabilité et les signaux SPa / SPv.
presence_threshold_pctSeuil minimal de RATIO_IN_POSITION.

Condition de tradabilité Game :

BMD >= tradability_threshold
SUM_SLOPE >= slope_threshold
RATIO_IN_POSITION >= presence_threshold_pct

Référence rapide

Les métriques numériques détaillées sont visibles dans les exports DailyMetrics. Les codes de signaux sont visibles dans le champ alerts, les alertes quotidiennes, les backtests et les Games.
