Parcours S&P500 historique

Utilise les outils historiques S&P500 existants : initialisation du référentiel, récupération de la composition historique via EODHD, création des actions manquantes et préparation des prix. Lancez les étapes dans l'ordre. Aucune étape ne lance toute la chaîne automatiquement.

Parcours CSI300 historique via CSV

  1. Générer le CSV CSI300 avec l'outil unliftedq vers StockAlert.
  2. Importer le fichier CSV CSI300 dans le bloc d'import staff-only.
  3. Créer / mapper les actions depuis les memberships CSV.
  4. Préparer les prix CSI300 via EODHD.
  5. Vérifier la readiness CSI300.
  6. Lancer le backtest avec le mode “CSI 300 historique dynamique — via CSV”.

Aucun constituant CSI300 n'est récupéré depuis EODHD. Aucune composition actuelle n'est utilisée comme fallback. EODHD sert uniquement à récupérer les prix des tickers issus du CSV importé.

Note CSI300 : les filtres GM_market et GM_sector restent non supportés en V1 tant que les benchmarks marché/secteur Chine ne sont pas configurés. Les benchmarks US type SPY/ETF sectoriels ne sont pas utilisés pour CSI300.
