Légende & définitions

Important : les grandeurs G, S_G_N, BT, BMJ et BMD sont stockées comme des ratios (ex : 0.01 = +1%). L'interface les affiche en pourcentage.

  • ratio_p (%) : indicateur utilisé pour décider si le ticker est tradable. {% if bt.include_all_tickers %} (bypass activé : l'éligibilité n'est pas conditionnée par ratio_p ≥ X) {% else %} (tradable si ratio_p ≥ X) {% endif %} Il est calculé par tes jobs d'alertes.
  • Action : BUY / SELL / FORCED_SELL (clôture forcée fin de période). SELL est prioritaire si BUY et SELL le même jour.
  • G : gain du coup clôturé ce jour (uniquement les jours de SELL). G = (prix_vente - prix_achat) / prix_achat.
  • N : nombre total de coups clôturés (BUY+SELL complétés) jusqu'à la date.
  • S_G_N : moyenne cumulée des gains par coup : S_G_N = (ΣG) / N.
  • BT : bénéfice total cumulé (somme des gains) : BT = ΣG (équivalent à S_G_N × N).
  • TRADABLE_DAYS : nombre de jours tradables dans la période. {% if bt.include_all_tickers %} (bypass activé : tous les jours de cotation de la période) {% else %} (ratio_p ≥ X) {% endif %}
  • TRADABLE_DAYS_NOT_IN_POSITION : nombre de jours tradables où l'on termine la journée hors position (shares = 0).
  • TRADABLE_DAYS_IN_POSITION_CLOSED : nombre de jours tradables où l'on termine la journée en position (shares > 0).
  • RATIO_NOT_IN_POSITION : 100 × TRADABLE_DAYS_NOT_IN_POSITION / TRADABLE_DAYS.
  • RATIO_IN_POSITION : 100 × TRADABLE_DAYS_IN_POSITION_CLOSED / TRADABLE_DAYS.
  • BMJ : bénéfice moyen par jour tradable hors position : BMJ = BT / TRADABLE_DAYS_NOT_IN_POSITION.
  • BMD : bénéfice moyen par jour tradable en position : BMD = BT / TRADABLE_DAYS_IN_POSITION_CLOSED.
  • BT portefeuille : rendement global du portefeuille sur l'investi final : (Equity finale - Investi final) / Investi final.
  • BMJ portefeuille : rendement portefeuille moyen par jour investi : BT portefeuille / NB_DAYS.
  • NB_DAYS : nombre de jours de la courbe portefeuille où invested > 0.
  • Cash : cash restant pour ce ticker/ligne après achats/ventes (arrondi au nombre entier d'actions).
  • Shares : nombre d'actions détenues en fin de journée.
  • TOTAL_PNL_AMOUNT : somme des gains et pertes en montant sur l'ensemble du portefeuille.
  • TOTAL_RETURN_ON_CAPITAL : rendement global du portefeuille = TOTAL_PNL_AMOUNT / capital total.
  • FINAL_EQUITY : capital final = capital initial + P&L total.
  • TOTAL_GAIN_AMOUNT : somme des trades gagnants en montant.
  • TOTAL_LOSS_AMOUNT : somme des trades perdants en montant.
  • AVG_TRADE_AMOUNT : gain/perte moyen(ne) par trade clôturé, en montant.
  • PROFIT_FACTOR_AMOUNT : rapport total gains / |total pertes|.
  • MAX_GAIN_AMOUNT : meilleur trade en montant.
  • MAX_LOSS_AMOUNT : pire trade en montant.
  • TOTAL_TRADES : nombre total de trades clôturés.
  • WIN_RATE_AMOUNT : pourcentage de trades gagnants rapporté au nombre total de trades, y compris les trades flats.
  • max_drawdown_amount : drawdown maximal du portefeuille en montant.
  • PNL_AMOUNT : gain/perte total(e) en montant pour le ticker / la ligne sélectionnée.
  • WIN_TRADES : nombre de trades gagnants pour le ticker / la ligne.
  • LOSS_TRADES : nombre de trades perdants pour le ticker / la ligne.
